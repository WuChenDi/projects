# FEAT-021 flnk — link display title (management name)

- **status**: completed
- **priority**: P2
- **owner**: bkd:claude-code
- **createdAt**: 2026-06-29

## Description

Short links are currently identified in the dashboard only by `slug` + the
optional free-form `comment`. Add a first-class **display title** so a link can
carry a human-readable management name (e.g. "Spring campaign landing") shown as
the primary label in the link list and editor.

This is distinct from the two existing title-like fields and must not reuse them:

- `links.comment` — a free-form note (unchanged).
- `config.title` — the **OG** title served to social crawlers (unchanged).
- `links.title` — **new** management display name.

## Schema

- `links.title`: `text('title').notNull().default('')`. Migration via
  `drizzle-kit generate` (never hand-authored).

## Work

- `database/schema.ts` — add `title` column; `pnpm db:gen` → apply.
- `schemas/link.ts` — add top-level `title: z.string().trim().max(256).optional()`
  to `CreateLinkSchema` (inherited by Edit/Upsert).
- `lib/links.ts` — thread `title` through `createLink` / `updateLink` /
  `importLinks` / export; default `''`.
- Dashboard link list — show `title` as the primary label, slug as secondary
  (fallback to slug when title empty). Editor — add a Title input.

## Acceptance Criteria

- Migration generated by drizzle-kit and applied locally; existing rows default
  to `''`.
- Create / edit / import round-trip `title`; export includes it.
- List shows title as primary label, falls back to slug when empty.
- `tsc` + biome clean.

## Out of Scope

- Renaming `comment` or `config.title`; any analytics on title.
