## Description

Replace Sink's single shared `SITE_TOKEN` Bearer-gate with **better-auth**
social login (**Google + GitHub only**, login == registration), backed by the
existing Drizzle (D1 / libsql dual-driver) database. Mirror the flox pattern,
adapted for OpenNext (node runtime, `getCloudflareContext()`). No email/password.

## Acceptance Criteria

- `better-auth` mounted at `/api/auth/[...all]`; Google + GitHub sign-in works
  (first sign-in auto-creates the user).
- All `/api/*` dashboard routes are gated by a valid session (not a Bearer token).
- Dashboard pages redirect unauthenticated users to the login page.
- `SITE_TOKEN` removed from code, `env.ts`, `wrangler.jsonc`, `.env.example`, README.
- Login page is two social buttons (i18n en/zh), no hardcoded Chinese in code.
- Auth tables added to `schema.ts`; init migration regenerated by drizzle-kit
  (existing init deleted + regenerated, not hand-written).
- `next build` + biome clean.
